# OFocus Documentation

OmniFocus SDK + CLI + MCP server for programs and AI agents. This folder is the map; the
root [`README.md`](../README.md) is the quick-start.

## Start here

- **[architecture.md](./architecture.md)** — how OFocus works today: package graph, the
  OmniJS bridge, the descriptor registry, query/error/output models, and how to add a command.
- **[agent-guide.md](./agent-guide.md)** — for AI agents: what's in the specs, when to consult
  them, and how to spot dissonance between spec, plan, and code. (Canonical source for
  `CLAUDE.md` / `AGENTS.md` / Copilot / Cursor guidance.)

## Conventions

- Design specs live in `docs/superpowers/specs/`; plans in `docs/superpowers/plans/`
  (written by the brainstorm → spec → plan workflow). This index keeps them discoverable.
- Keep specs current: a behavior change that leaves a spec asserting the old contract is a
  defect (see [agent-guide.md](./agent-guide.md) §4).
